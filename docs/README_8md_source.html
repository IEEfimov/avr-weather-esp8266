9 ### Short description
10 
11 This project tends to implement:
12 
13 * ESP8266 module messaging from AVR controller
14 * Basic Http request + JSON parsing
15 * LCD support
16 
17 ### What does it do?
18 
19 * Boots up (if you press any key) the avr enters
20 
21 `settings mode` and asks for wifi creditentials
22 
23 * If no key was pressed during boot then normal procedure takes place
24 
25 * AVR tries to connect to the wifi
26 
27 * Then connects to `openweathermap.com`
28 
29 * Asks politely for weather data for `Warsaw`
30 
31 * Then fetches `JSON` with the most trivial ways
32 
33 * Displays the data
34 
35 * And sometimes refreshes the collected data :)
36 
37 ### Pinout
38 
39 The AVR connections are basic and listed below:
40 
41 
42 
43 * `(ATMEGA32) PA0 -> D4 (LCD)`
44 
45 * `(ATMEGA32) PA1 -> D5 (LCD)`
46 
47 * `(ATMEGA32) PA2 -> D6 (LCD)`
48 
49 * `(ATMEGA32) PA3 -> D7 (LCD)`
50 
51 * `(ATMEGA32) PA4 -> RS (LCD)`
52 
53 * `(ATMEGA32) PA5 -> RW (LCD)`
54 
55 * `(ATMEGA32) PA6 -> E (LCD)`
56 
57 * `(ATMEGA32) RXD -> TX (ESP8266)`
58 
59 * `(ATMEGA32) TXD -> RX (ESP8266)`
60 
61 * `(ESP8266) EN -> (ESP8266) VCC`
62 
63 
64 
65 
66 
67 And of course `V0`, `VCC`, `GND`, `VSS`, `VDD`, `A`, `K` but they are trivial.
68 
69 And not mentioned in layout two buttons for settings screen:
70 * `PD6 -(BUTTON)-> GND`
71 
72 * `PD7 -(BUTTON)-> GND`
73 
74 
75 
76 
77 
78 In settings mode:
79 
80 * `PD6` is used have next character functionality.
81 
82 * `PD7` changes current character.

87 ### About building
88 
89 To build the project do the following:
90 
91 * Create release folder e.g. `mkdir release`
92 
93 * Go into that folder `cd release`
94 
95 * Run CMAKE to generate needed makefiles `cmake ..`
96 
97 * Build app using make `make`
98 
99 * Optionally update documentation throught `make doc`
100 
101 * Optionally setup manually the needed fuse bits using `avrdude`!
102 
103 * Flash controller with `make upload`
104 
105 * Optionally clean build with `make clean`
106 
107 
108 
109 
110 The main controller config is placed in `CMakeLists.txt` file.
111 
112 
113 
114 Please change these as you wish:
115 
116 ```bash
117 # Variables regarding the AVR chip
118 set(MCU atmega32)
119 set(F_CPU 8000000)
120 set(BAUD 9600)
121 set(PROG_TYPE usbasp)
122 set(PROG_ARGS )
123 ```
124 
125 
126 `BAUD` is not really needed and `PROG_ARGS` are additional paramters for avrdude.
127 
128 
129 
130 Basic example config for `Atmega328P-PU` running (by default) at 1MHz:
131 
132 ```bash
133 # Variables regarding the AVR chip
134 set(MCU atmega328p)
135 set(F_CPU 1000000)
136 set(BAUD 2400)
137 set(PROG_TYPE usbasp)
138 set(PROG_ARGS )
139 ```

141 ### Some source code
142 
143 ```c
144 
145 int main(void) {
146 
147  // Init UART
148  uart_init(UART_BAUD_SELECT(2400, F_CPU));
149 
150  // Init buttons
151 
152  //
153  // ...
154  //
155 
156  // Init LCD
157  lcd_init(LCD_DISP_ON);
158  lcd_gotoxy(0, 0);
159  lcd_clrscr();
160 
161  // Init wifi and connect to the desired network
162  lcd_clrscr();
163  lcd_puts("Initializing...");
164  _delay_ms(1000);
165  wifi_init();
166 
167  //
168  // Check if any button was pressed
169  // then go into settings mode :)
170  //
171 
172  //
173  // ...
174  //
175 
176  //
177  // Normal boot sequence
178  //
179  normal_boot:
180 
181  // Display greeting on lcd
182  lcd_puts("Hello");
183  _delay_ms(1000);
184  lcd_clrscr();
185  _delay_ms(1500);
186 
187 
188  lcd_clrscr();
189  lcd_puts("Conecting...");
190  _delay_ms(1000);
191 
192  lcd_clrscr();
193  lcd_puts("ID: ");
194  lcd_puts(USSID);
195  lcd_puts("\nPASS: ");
196  lcd_puts(PASSWD);
197  _delay_ms(1500);
198 
199  wifi_connect(USSID, PASSWD);
200 
201  while(1) {
202  uart_flush();
203 
204  // Connect to openweathermap
205  wifi_link_open("TCP", "api.openweathermap.org", 80);
206 
207  // Request data from REST api
208  wifi_send("GET /data/2.5/weather?q=warsaw&APPID=cf24c94ac2550178e2ea4e970cd0f416 HTTP/1.1\r\nHost: api.openweathermap.org\r\nConnection: keep-alive\r\nCache-Control: max-age=0\r\n\r\n\r\n");
209  }
210 
211  return 0;
212 }
213 ```
